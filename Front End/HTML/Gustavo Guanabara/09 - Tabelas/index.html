CRIAR TABELAS NO HTML

                    Anterior

As Tabelas no HTML devem ser usadas para exibir dados tabulares - foram projetadas para isso. Infelizmente, muitas pessoas se acostumaram a usar tabelas HTML para diagramar páginas web, por exemplo, uma linha para conter o cabeçalho, outra linha para conter as colunas de conteúdo, uma linha para conter o rodapé, etc...

Existe 3 maneiras de criar tabelas:

  1. Tabela Simple

A tabela simple é feita por linhas e colunas sem ter a necessidade de especificar o cabeçário ou o corpo da tabela. Para criar uma tabela simples usa-se essa estrutura: 

            Dentro da tag Main
            ~~~~~~~~~~~~~~~~~~~~~
            <table>
                <tr>
                    <td> ou </td>
                    <th> ou </th>
                </tr>
            </table>

Na formatação no CSS, colocamos seguintes parametros: 

            Formatação internal
            ~~~~~~~~~~~~~~~~~~~
            <style>
                <th, td>
                {
                    border: 1px solid black (As linhas que enquadra a tabela)
                }
            </style>
  • A tag table
    • - Serve para colocar a tabela, entretanto dentro da tabela temos linhas e colunas.

  • A tag tr
    • - abreviação tr significa Table row - Linha da tabela - o table row serve para colocar AS LINHAS dentro da tabela.

  • A tag th
    • - abreviação th significa Table header - tabela para cabeçário da linha - o table header serve para colocar OS DADOS DO CABEÇÁRIO DA LINHA dentro da tabela.

  • A tag td
    • - abreviação td significa Table data - Dado da tabel - o table dado serve para colocar OS DADOS DA LINHA dentro da tabela.

Se observar que na tabela 01, as linhas das bordas não estão interligado, para melhorar a nossa formatação, colocaremos seguintes formatação. 

            Formatação internal
            ~~~~~~~~~~~~~~~~~~~
            <style>
                <body>
                {
                    border-collapse: collapse; - serve para juntar as bordas
                }
                <th, td>
                {
                    border: 1px solid black (As linhas que enquadra a tabela)
                }
            </style>

  1. Tabela Complexa

A tabela complexa ou composta é composto de cabeçário, copro e rodapé, neste modelo, é nécessário especificar e melhor organizar a tabela. Para criar uma tabela complexa ou composta usa-se essa estrutura: 

            Formatação internal
            ~~~~~~~~~~~~~~~~~~~
            <table>

                Para cabeçário
                --------------
                <thead>
                    <tr>
                        <td> ou <th>
                        </td> ou </th>
                    </tr>
                </thead>

                Para corpo
                ----------
                <tbody>
                    <tr>
                        <td> ou <th>
                        </td> ou </th>
                    </tr>
                </tbody>

                Para rodapé
                -----------
                <tfooter>
                    <tr>
                        <td> ou <th>
                        </td> ou </th>
                    </tr>
                </tfooter>
            </table>

        Veja o exemplo 03
  • A tag thead
    • - abreviação thead significa Table head - Cabeçário da tabela - o table head é cabeçário da tabela, ao contrário do th - que é o cabeçário da LINHA. dentro da cabeçário podemos ter tr, th e td.

  • A tag tbody
    • - abreviação tbody significa Table body - Corpo da tabela - o corpo da tabela é meio ou barriga da tabela. dentro da cabeçário podemos ter tr, th e td.

  • A tag tfooter
    • - abreviação tfooter significa Table footer - Rodapé da tabela - o rodapé da tabela é o fim da tabela. dentro da cabeçário podemos ter tr, th e td.

  1. Tabela Mesclada

O significado da palavra Mesclar - Misturar, logo, a tabela mesclada é uma tabela mista onde tem mistura complexa de linha e coluna. A tabela mesclada deve fazer parte da estrutura da tabela complexa. Para criar uma tabela complexa ou composta usa-se a esturuta da tabela complexa e com seguintes comandos:

<tag(thead, tbody, tr, th ou td) rowspan ="número de linha para juntar" > - Usa-se o comando rowspam - para mesclar ou juntar 2 linhas.

<tag(thead, tbody, tr, th ou td) colspan = "número de coluna a juntar"> - Usa-se o comando colpam - para mesclar ou juntar 2 coluna.

Outras configurações

Existe várias outras configurações podemos usar nas tabelas, algumas delas são essas:

  • Caption
Caption pode ser traduzido rubrica ou legenda da tabela, ela serve para colocar o nome ou título acima de qualquer tabela.

Nota que pouca importa onde pode colocar caption dentro da tabela ela sempre será no inicio da tabela.. 

            Configura-se a caption da seguinte maneira:
            <style>
                 <caption>O tema desejado</caption>
                <thead>
                    <tr, th, td>
                </thead>
                <tbody>
                    <tr,td>
                </tbody>
                <tfooter>
                    <tr, td>
                </tfooter>
            <style>
        Vejamos tabela 02

  • Escopo de títulos

Os escopos dos títulos são comandos que facilita o google ler o nosso site, vizualmente não há efeito direto mas se consideramos a facilidade que os escopos deixam o nosso site, vale apena tentar de colocar. Nota que os escopos serve SOMENTE PARA SEREM USADOS DENTRO DOS th a fim de identificar o tema da linha.. 

            Configura-se a caption da seguinte maneira:
            <style>
                <thead>
                    <th scope = "col">
                </thead>
                <tbody>
                    <th scope = "row">
                </tbody>
                <tfooter>
                    <thscope = "groupcol">
                </tfooter>
            <style>

        Existe 4 tipos de escopos
        scope = "col" - serve para identificar as colunas.
        scope = "goupcol" - serve para identificar os grupos colunas.
        scope = "row" - serve para identificar as linas.
        scope = "grouprow" - serve para identificar groupos das linas.

  • Formatação das linhas ou Efeitos zebrados

Para formater linhas ou cria-se efeito zebrado de duas maneiras possiveis:

  1. Metodo de identificação das classes
    2.
    Para criar um efeito zebrado ou formatação das linhas usando a identificação do tipo class basta colocar a mesma identificação em cada linha das tag (tr ou th ou td). importante de notar, se a lista for extensa, será muito dificil exeturar por este modelo e também caso houver acrescimo de algumas linhas, será necessário formatar de novo a Configuração.


  2. Metodo do comando tag:nth-chuld(2,3,4,5.... n) ou tag:nth-chuld(odd)
    3. tag:nth-chuld(odd) - Essa Configuração é muito didático e importante porém é muito limitado, porque ele se basea somete nos números pares pulando 2 passos. caso o desenvolvidor quer pular de 5 à 5 passo deve usar essa Configuração 👉 tag:nth-chuld(5n)


  3. Metodo do comando tag:nth-chuld(2,3,4,5.... n-1) ou tag:nth-chuld(even)
    4. tag:nth-chuld(even) - Essa Configuração é muito didático e importante porém é muito limitado, porque ele se basea somete nos números ímpares pulando 2 passos. caso o desenvolvidor quer pular de 5 à 5 passo deve usar essa Configuração 👉 tag:nth-chuld(5n-1) 
                        Formatação internal
                    ~~~~~~~~~~~~~~~~~~~
                    <style>;
                    tbody > tr:nth-chuld(2n)
                        {
                            background-color: red;
                        }
                        
                    </style>

  • Formatação das Colunas

Para agrupar as linas usamos 3 metodos,portanto para agrupar colunas usaremos 2 metodos simples. Metodo de identificação das classe e Metodo do comando colgroup

  • Metodo de identificação das classes
    •
    Para criar um agrupamento das colunas usando a identificação do tipo class basta colocar a mesma identificação em cada coluna das tag (tr ou th ou td). importante de notar, se a lista for extensa, será muito dificil exeturar por este modelo e também caso houver acrescimo de algumas linhas, será necessário formatar de novo a Configuração.


  • Metodo do comando colgroup
    •

    A sua formatação é feita dentro da tabela, ao contrário de metodo anterio (Formatação das linhas), exetuta-se essa formatação da seguintes maneiras: 

                    <table>
                    <colgroup>
                        <col class ="nome da coluna que quer formatar" >
                        <col class ="nome da coluna que quer formatar">
                            .
                            . 
                            .
                        <col class ="nome da coluna que quer formatar">
                    <colgroup>
                (👉Aqte aqui agrupamos as colunas que queremos formatar, para formatar dpodemos usar formatação external ou internal)
                </thead>
                    <tr,th,td>
                <tbody>
                    <tr,th,td>
                </tbody>
                <tfooter>
                    <tr,th,td>
                </tfooter>
                <style>

                Formatação internal
                ~~~~~~~~~~~~~~~~~~~
                <style>
                    nome da coluna que quer formatar
                    {
                        background-color: red;
                    }
                    
                </style>

  • Cabeçalho fixo em tabelas grandes

Acontence que quando viewport for pequeno e a tebla for grande, os dados começa a flutuar no ecrã, isso dificulta muito a vida do usuário. Razão pela qual sentimos a necessidade de fixar o cabeçário no inicio para que os dados não flutuam nos pequenos viewport. Fixa-se o cabeçário de seguinte maneira: 

            Formatação internal
            ~~~~~~~~~~~~~~~~~~~
            <style>
                table
                {
                    position: relative;
                }
                thead > tr > th
                {
                    position: sticky;
                    top: 0; - Aqui so vai colar a palavra, razão pela qual acrescentamos o fundo.
                    background-color: A cor deve a mesma que se colocou no (thead, tfoot);
                }
            </style>

  • Tabelas responsivas

Para criar uma tabela responsivel, obrigatóriamente vamos precisar de uma formatação com tag div para ser de container e 

            <main>
                <div class ou id ="container">
                    <table>
                        A nossa tabela completa
                    </table>
                </div>
            </main>
            
            Formatação internal
            ~~~~~~~~~~~~~~~~~~~
            <table>
                . ou #div
                {
                    width: (50,60,70,90)vw - isto é opçional
                    overflow-x: auto;
                }
            <table>
